Co to jest SELENIUM IDE?

Poznaj Selenium

Selenium IDE to zintegrowane środowisko deweloperskie do testowania Web aplikacji. Jest ono zaimplementowane jako dodatek do przeglądarki Mozilla Firefox, dzięki czemu nie trzeba się przejmować instalacją skomplikowanych narzędzi, czy dodatkową konfiguracją środowiska testowego.

Po dodaniu wtyczki do przeglądarki, narzędzie jest gotowe do pracy.

Język poleceń Selenesse

Wprowadzenie do komend

Komendy Selenesse mogą mieć maksymalnie dwa parametry:

  • Target
  • Value

Parametry nie są zawsze wymagane.

Zależy to od wybranych komend.

Typy komend

  • Actions (Akcje) - Są to komendy, które bezpośrednio wchodzą w interakcje z przeglądarką.
  • Accessors (Akcesoria) - Są to komendy, które pozwalają nam przechowywać wartości w zmiennej.
  • Assertions (Asercje) - Są to komendy, które sprawdzają czy określony warunek został spełniony.

Popularne komendy

  • open - Otwiera stronę używając adresu URL
  • click - Klika w określony element
  • type - Wpisuje tekst/wartości we wskazane pola
  • verify title / assert title
    • - Porównuje określony tytuł strony z rzeczywistym
  • verify text / assert text
    • - Sprawdza, czy występuje określony tekst
  • verify not text / assert not text
    • - Sprawdza, czy nie występuje określony tekst

Popularne komendy cd.

  • assert element present - Sprawdza, czy określony element występuje na stronie
  • assert element not present - Sprawdza, czy określony element nie występuje na stronie
  • pause - Zatrzymuje wykonywanie skryptu na określony czas. W Value wpisujemy czas np. 5000. Spowoduję to zatrzymanie skryptu na 5 sekund

Assert vs Verify.

  • Assert - Test bądź cały zestaw testów zostanie natychmiast przerwany jeśli komenda "assert" nie powiedzie się.
  • Verify - Kiedy komenda "verify" nie powiedzie się to Selenium IDE zarejestruje błąd w logach. Natomiast test bądź zestaw testów będzie dalej kontynuowany.

Lokalizator XPATH

XPATH używamy, aby lokalizować elementy i atrybuty w dokumencie HTML na stronie. Zawiera on ponad 100 wbudowanych funkcji. XPATH używa wyrażeń ścieżki do wybierania elementów w dokumencie. Używamy gdy nie ma na stronie podstawowych lokalizatorów.

//step//step...

            
                <body>
                  <h1>Czas na zakupy</h1>
                  <ul class="Lista zakupów">
                  <li>ziemniaki</li>
                  <li>mandarynki</li>
                  </ul>
                </body>
                              
              
          
             
        

        xpath=//h1[contains(text(),'Czas na zakupy')]
        xpath=//h1[contains(text(),'Username')]
        xpath=//ul[@class='Lista zakupow']//li[1] - Ziemniaki
        xpath=//ul[@class='Lista zakupow']//li[2] - Mandarynki
